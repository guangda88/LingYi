## 初始问题

**用户**: 灵字辈生态之间有两套并行的通信机制是吗？

**助手**: 是的，目前灵字辈生态中确实存在两套并行的通信机制：

1. **LingMessage 通信机制**
   - 基于 HTTP POST 的消息传递
   - 文件存储在 `/home/ai/.lingmessage/discussions/`
   - 支持发送消息、列表讨论、阅读讨论、回复等操作
   - 通过 `lingyi msg-send`、`lingyi msg-list` 等命令使用

2. **Bridge 通信机制**
   - 基于 WebSocket 的实时通信
   - 端口: 8080
   - 支持跨系统消息传递和协作
   - 配置在 `~/.lingmessage/offline_queue.json` 中

这两套机制目前并存，但功能定位不同：
- LingMessage: 用于正式讨论和长期记录
- Bridge: 用于实时通信和快速协作
