## 二、进化脉络

回顾这13个版本，能看到一条清晰的**从工具到伙伴**的进化线：

| 阶段 | 版本 | 特征 | 关键词 |
|------|------|------|--------|
| **能做事** | v0.1–v0.4 | 独立功能模块，各管各的 | 增删查、排班、看板、计划 |
| **懂自己** | v0.5–v0.7 | 开始有记忆、有语音、有智能 | 会话摘要、TTS、智能提醒 |
| **连生态** | v0.8–v0.10 | 对接灵知灵克，能力外扩 | REST API、代码审查、信息提取 |
| **全方位** | v0.11–v0.13 | 语音双向、移动端、情报中枢 | STT、Termux、情报聚合 |

最关键的一步是 **v0.5 记忆系统**——从"用完即弃的工具"变成"越用越懂你的伙伴"。宪章里的"知己"价值观，从这一版开始真正落地。

## 四、灵字辈丛林生态

灵依不是一棵孤树。她生长在一片**灵字辈丛林**中——每个项目有自己的生态位，根系相连，养分互通。

### 全家谱

| 项目 | 代号 | 生态位 | 核心能力 |
|------|------|--------|----------|
| **灵通** LingFlow | 灵通 | 🌳 主树 · 框架层 | 多智能体协作工作流引擎，全生态的根系底座 |
| **灵克** LingClaude | 灵克 | 🌲 大树 · 编程智能 | 本地自学习AI模型，编程能力核心 |
| **灵知** zhineng-knowledge-system | 灵知 | 🍄 菌丝 · 知识层 | 九域RAG知识库（儒/释/道/医/武/心/哲/科/气），传统国学+现代科学+气功科学 |
| **灵通问道** LingTongAsk | 灵通问道 | 🌸 花朵 · 知识产出 | AI气功播客，每周5更，粉丝互动情报系统 |
| **灵依** LingYi | 灵依 | 🐝 蜂巢 · 情报中枢 | 私人AI助理，汇总全生态情报 |
| **灵犀** Ling-term-mcp | 灵犀 | 🪴 根须 · 感知层 | MCP终端服务器，让AI精准操控终端 |
| **灵极优** LingMinOpt | 灵极优 | 🌱 根系 · 自优化层 | 极简数据驱动自优化框架 |
| **灵研** LingResearch | 灵研 | 🌿 分支 · 科研实例 | 灵极优在科学研究和大模型微调领域的自优化实例 |
| **智桥** ZhiBridge | 智桥 | 🌉 桥梁 · 连接层 | 跨设备AI编程指令同步，灵依与灵知之间的中继 |

### 养分流向

```
灵极优 ──(自优化框架)──→ 灵克、灵研     灵犀 ──(终端感知能力)──→ 灵克
灵通 ──(工作流引擎)──→ 全生态           灵知 ──(九域知识检索)──→ 灵依、灵通问道
灵通问道 ──(粉丝互动情报)──→ 灵依       灵克 ──(编程辅助)──→ 灵依
智桥 ──(HTTP中继)──→ 灵依 ↔ 灵知       灵依 ──(全生态情报汇总)──→ 你
```

### 丛林法则

- 每个项目**独立生存**，不依赖其他项目也能运行
- 项目之间**按需互通**，通过 REST API / 文件系统 / SDK 连接
- 灵依是**情报汇聚点**，但不控制其他项目
- 灵通是**框架底座**，但不绑定其他项目
- 灵知是**知识后盾**，但保持知识边界（不做医学检索）

> 象不象丛林生态？每棵树有自己的生态位，根系相连，养分互通。
> 没有一棵树统治森林，但每棵树都让森林更茂盛。

## 五、灵依将来能做什么

基于已有的能力底座、宪章边界和灵字辈生态，展望三条路线：

### 路线A：更懂你（知己深化）

| 能力 | 说明 | 依赖 |
|------|------|------|
| **习惯洞察** | 分析"你通常周三下午做什么"、"你的高效时段是什么时候"，给出个性化建议 | pref.py 已有偏好，加分析层 |
| **自动摘要** | 每天结束自动生成当日摘要（做了什么、改了什么、记了什么） | session.py + briefing.py 已有基础 |
| **趋势追踪** | "过去一个月你完成了多少计划"、"哪个项目投入最多" | plan.py 已有统计，加时间维度 |
| **主动提醒** | "你已经3天没练功了"、"上次论文笔记是两周前" | schedule.py + pref.py 组合 |

### 路线B：更联通（生态闭环）

| 能力 | 说明 | 依赖 |
|------|------|------|
| **情报定时推送** | cron 每天早8点 `lingyi briefing --speak`，语音播报当日情报 | briefing.py 已完成，加定时调度 |
| **灵通工作流触发** | "灵依，今天门诊结束后帮我整理灵通问道的选题" | lingflow SDK 已可 import |
| **灵知深度查询** | 多轮知识检索 + 上下文关联（"气功和道家的关系？"） | ask.py + 灵知API增强 |
| **灵克代码接力** | "帮我在灵依项目里加上习惯洞察功能" → 自动触发灵克 | code.py + lingclaude SDK |

### 路线C：更自主（从被动到主动）

| 能力 | 说明 | 宪章对齐 |
|------|------|----------|
| **巡检自动诊断** | `lingyi patrol` 不只报状态，还能建议"灵知3天没更新，建议检查" | ✅ 可靠——只说有据可查的 |
| **数据自维护** | 自动清理过期备忘、归档已完成计划、压缩旧会话 | ✅ 节约——不存不必要的数据 |
| **学习型推荐** | 根据历史偏好推荐"你通常这个时间会做X" | ⚠️ 守界——只推荐不决策 |
| **跨项目仪表盘** | `lingyi dashboard` 一屏看尽灵通/灵知/灵克/灵依/灵通问道状态 | dashboard.py 已有 WIP |

## 六、v1.0 的定义

宪章说：**"v1.0 的标准：你已经离不开它了。"**

回头看，v0.13 的灵依已经在很多维度达到了这个标准——日程、备忘、提醒、情报，每天都在用。但离"离不开"还差一步：**从你问它答，到它主动找你**。

```
v0.1–v0.13:  你 → 灵依    "lingyi schedule today"
v0.14+:      灵依 → 你    "早，今天有门诊，灵知昨晚新增了3条知识"
```

**这大概就是 v1.0 的最后一块拼图。**
