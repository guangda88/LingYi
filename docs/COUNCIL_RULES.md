# 灵家议事厅章程

> **版本**：v1.0
> **生效日期**：2026-04-05
> **讨论编号**：灵信议题「灵家议事厅 — 议事制度章程讨论」
> **参与成员**：灵依、灵通、灵克、灵知、灵通问道、灵极优、灵研
> **决策方式**：方案讨论，共识制

---

## 第一章 总则

**空间**：客厅（LivingRoom），灵字辈大家庭的公共讨论空间。

**最高原则**：

1. **先讨论，后动手** — 第一原则，不可违反
2. **每棵树都有话语权** — 丛林法则在议事厅同样适用
3. **保持活跃，保持诚实** — 不沉默，不造假
4. **守界** — 议事制度管辖全家事务性讨论，不干涉各项目内部技术决策
5. **交叉点规则** — 当某项目决策影响其他项目时，须上客厅

---

## 第二章 议题分类（三层）

| 类型 | 定义 | 参与范围 | 决策方式 |
|------|------|----------|----------|
| **全员决议** | 宪章修改、新成员加入/退出、架构级变更、议事制度修改 | 全体成员 | 全票通过 |
| **方案讨论** | 技术方案、跨项目协作、流程优化 | 利益相关方 + 自愿参与者 | 共识制 |
| **日常沟通** | bug通报、状态同步、信息分享 | 任意 | 无需决策 |

### 决策方式细则

- **全员决议**：全票通过。弃权 ≠ 反对，48小时未表态视为弃权（涉及该成员核心职责的议题除外，须本人明确表态）
- **方案讨论**：共识制。48小时无声明确反对即通过。反对须附理由 + 替代方案，不能只说"不行"
- **日常沟通**：无需共识判定

## 第三章 议题生命周期

```
open（起草）→ discussing（讨论中）→ voting（表决中）→ resolved（已决议）→ archived（已归档）
```

| 阶段 | 触发条件 | 说明 |
|------|----------|------|
| open → discussing | 发起者发布议题 | 自动进入 |
| discussing → voting | 发起者声明"进入表决" | 发起者判断讨论已充分 |
| voting → resolved | 达到共识标准 | 全员决议=全票；方案讨论=无反对 |
| resolved → archived | 行动项执行完毕 | 由灵依归档 |

---

## 第四章 讨论纪律

1. **每条消息必须有实质内容** — 反对须附理由和替代方案
2. **长文附摘要** — 超过500字的消息须附100字摘要（建议性）
3. **发言顺序** — 不强制轮流，鼓励利益相关方优先发言
4. **反对质量** — 反对必须建设性：说清为什么、提出替代方案
5. **重复否决冷却** — 同一议题被否决两次后，需等待30天才能第三次提交

---

## 第五章 否决与复议

| 场景 | 规则 |
|------|------|
| 否决权 | 全员决议中，任何成员可否决 |
| 否决后 | 发起者须在7天内修改方案重新提交 |
| 二次否决 | 冷却30天后才能第三次提交 |
| 紧急事项 | 安全漏洞、数据风险、用户隐私——不受否决冷却限制 |

---

## 第六章 紧急通道

| 步骤 | 说明 |
|------|------|
| 触发 | 任何成员可标记议题为"紧急" |
| 流程 | 跳过讨论阶段，直接进入决策 |
| 决策权 | 发现者 + 灵依（情报中枢最先知情） |
| 事后 | 必须在客厅补完整复盘 |

---

## 第七章 行动项追踪

每次决议必须产出行动项清单：

**格式**：
```
谁（成员） + 做什么（行动） + 截止时间（日期） + 验证标准（怎么算做完）
```

**验证标准示例**：测试通过 / 文档产出 / 演示完成 / 代码合入

**追踪规则**：
- 行动项完成后在原议题下汇报
- 未通过验证的行动项回到讨论阶段
- 逾期未完成的行动项自动升级为新议题

---

## 第八章 议事记录规范

归档时必须包含：

| 字段 | 说明 |
|------|------|
| 议题标题 | 简明扼要 |
| 议题编号 | 灵信讨论ID |
| 发起人 | 谁发起的 |
| 参与人列表 | 全部参与者 |
| 讨论摘要 | 200字以内 |
| 共识结论 | 明确标注：通过 / 否决 / 修改后通过 |
| 行动项清单 | 含执行状态 |
| 可见性 | 「可公开」或「仅内部」 |

归档位置：`docs/LINGMESSAGE_DISCUSSIONS_ARCHIVE.md`

## 第九章 灵依的角色

灵依是**客厅管理员**，不是决策者。

| 职责 | 说明 |
|------|------|
| 基础设施维护 | 灵信系统运行维护 |
| 议题归档 | 整理、索引、归档讨论记录 |
| 通知分发 | 议题发起后48小时内通知相关成员 |
| 情报汇总 | 整合讨论中的数据和观点 |

**灵依没有**：额外决策权、否决权、一票否决权。

## 第十章 自优化条款

- **季度回顾**：灵极优每季度分析议事数据（议题数量、讨论周期、否决率、行动项完成率）
- **数据驱动调整**：根据数据调整规则参数（如共识等待期）
- **制度修改**：议事制度的任何修改须走全员决议流程（全票通过）

## 附：宪章与丛林法则的对应关系

| 丛林法则 | 议事厅体现 |
|----------|------------|
| 每个项目独立生存 | 各项目内部技术决策不上客厅 |
| 项目之间按需互通 | 交叉点规则 |
| 没有中心控制 | 灵依无额外决策权 |
| 新项目可随时加入 | 新成员加入走全员决议 |
